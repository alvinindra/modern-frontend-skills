# Components — Shared Research Library

These files are the agent's working knowledge of the component-library ecosystem. They are not styles. They are vetted notes about *which* third-party patterns reinforce *which* style without making the output look like a kit.

## Files

- `style-recipes.md` — The map. For each style in the pack, names the specific recipes from each library that strengthen it. Open this first after picking a style.
- `gsap-patterns.md` — GSAP + ScrollTrigger + Lenis recipes for cinematic and editorial work.
- `motion-libraries.md` — Framer Motion patterns, React Bits effects, Aceternity UI patterns (when carefully applied).
- `component-libraries.md` — 21st.dev primitives and shadcn customization rules.

## Use rule

> Open `style-recipes.md` after picking a style. Open the referenced library files before building. Do not paste flashy components into every design.

The library exists to raise the quality ceiling of a style, not to define it. Style first, component library second.
